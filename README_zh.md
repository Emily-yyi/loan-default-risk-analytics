# 贷款违约风险分析项目

本项目基于 Lending Club 贷款数据，围绕贷款审批策略、借款人风险分群和违约预测建模展开分析，目标是辅助金融机构在“控制坏账风险”和“保持合理审批通过率”之间做出更好的决策。

## 项目概览

本项目从信贷风控和业务决策角度分析消费贷款申请数据。整体流程包括：

- 对比不同贷款审批策略的风险表现
- 基于贷前可观测变量进行借款人分群
- 构建贷款违约预测模型
- 将模型输出转化为自动通过、人工复核、拒绝三类决策建议

## 业务问题

1. 保守型与激进型贷款审批策略，哪一种更能降低坏账误批风险？
2. 是否可以基于借款人的收入、负债、利率和信用行为等变量识别不同风险客群？
3. 深度学习模型是否比逻辑回归基线模型更适合该类结构化信贷数据？
4. 如何将模型预测分数转化为可执行的审批 / 复核 / 拒绝策略？

## 分析方法

- 使用坏账误批率作为核心风控指标，对比两种审批策略
- 使用双比例检验比较策略表现差异
- 使用层次聚类和 K-means 对借款人进行风险分群
- 进行特征工程，包括收入负担、循环额度使用、信用历史和缺失值标记等变量
- 使用 PyTorch 构建 DNN 违约预测模型
- 使用逻辑回归作为基线模型进行对比
- 进行阈值调优和分群层面的模型效果评估

## 关键结果

- 分析约 50,000 条 Lending Club 贷款记录，整体坏账率约为 15.6%。
- 以坏账误批率为核心指标，对比保守型审批策略和激进型审批策略。
- 保守型策略将坏账误批风险从约 42.2% 降低至约 0.6%，但审批通过率明显更低，体现了风控与业务增长之间的取舍。
- 识别出三类借款人风险群体：
  - 低风险客群：利率、DTI 和循环额度使用率较低，坏账率约 10.4%。
  - 高杠杆客群：收入较高，但债务负担更重，坏账率约 17.1%。
  - 高循环额度使用 / 财务压力客群：循环额度使用率较高，坏账率约 18.0%。
- PyTorch DNN 与逻辑回归模型的 ROC-AUC 均约为 0.67，说明在该结构化表格数据场景下，简单且可解释的基线模型具有较强竞争力。
- 基于模型预测概率提出三段式审批建议：低风险自动通过，中风险人工复核，高风险拒绝。
